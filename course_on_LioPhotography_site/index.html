The big picture

How the website is put together

You don't need the technical details, but a simple mental model makes everything else click.

The three moving parts

PartWhat it is
Your computerWhere the photos and website files live. You add photos and run one command here.
GitHubAn online “save vault” that stores a copy of the whole site.
NetlifyThe web host. Whenever GitHub updates, Netlify automatically rebuilds the live site within ~2 minutes.

So the flow is always: change on your computer → send to GitHub → Netlify publishes it.

Two kinds of photo information

Every photo can carry information in two layers that combine automatically:

  • Automatic layer — camera settings, date, and (sometimes) GPS read straight from the photo file's EXIF data. You don't type these.
  • Your layer — title, story, location, collection, keywords that you add in the editor. Your layer always wins if both have the same field.
Why this matters: re-running the build never erases a story you wrote. Your words are safe.

Core task

Uploading new photos

This is the most common job. Done right, the site reads most information automatically.

Step 1 — Export from Lightroom / Photoshop with metadata

This single setting is the secret to letting the site fill in camera details, keywords, and GPS for you.

Lightroom Classic

  • Add Title, Keywords, and Location/GPS while culling (Metadata & Keywording panels; GPS in the Map module).
  • On export → Metadata panel → set Include: “All Metadata”.
  • Uncheck “Remove Location Info” (so GPS is kept).
  • Resize long edge ~3840 px, colour space sRGB, quality ~90.

Photoshop

  • Add info via File → File Info (Title, Description, Keywords).
  • Save a Copy → JPEG keeps all metadata (best). Avoid “Export As” (it strips data).

Step 2 — Put the photos in the right folder

Inside the project folder you'll see folders like photos_Landscape, photos_Astrophotography, etc. Drop your new JPEGs into the matching one. A new folder named photos_Something automatically becomes a new gallery.

Naming tip: filenames can be anything (spaces and brackets are fine). Keep the original camera names or rename — your choice.

Step 3 — Run the build command

Open a terminal in the project folder and run:

python3 generate_manifest.py

This makes the optimized image sizes, reads the EXIF/keywords, and refreshes the galleries, search, and map. (Full detail in the next lesson.)

Step 4 — Publish

Send the changes live (covered in the “Publishing” lesson). That's it — your photos are now in their gallery, optimized, and searchable.

Remember: a freshly uploaded photo is already viewable and findable by its gallery name. Adding a title/story/keywords (next lessons) just makes it richer.

Behind the scenes

The build command — generate_manifest.py

One command does a lot of work for you. You run it after any change to photos.

python3 generate_manifest.py

What it does automatically

  • Creates web-friendly sizes of each photo (small for grids, large for full view) in modern formats (AVIF/WebP/JPEG) inside a hidden _web folder.
  • Reads each photo's EXIF (camera, lens, exposure, date) and GPS, and any keywords/title embedded by Lightroom.
  • Updates the galleries list, the search index, the map pins, and the sitemap for Google.
  • It's incremental — only new or changed photos are processed, so re-running is quick.
If python3 isn't found: try py generate_manifest.py (Windows) — or ask whoever set up the computer to confirm Python is installed.

When to run it

  • After adding, removing, or replacing photos.
  • After adding photos to the prints/ folder.
  • You do not need to run it after only editing photo details in the editor (that updates a different file).

Core task

Adding details with the metadata editor

This is where photos get their title, story, location, collection, and keywords — the things that power story pages, the map, collections, and search.

Open the editor

Double-click meta-editor.html in the project folder (it opens in your browser, works offline). Or open the live version at /meta-editor.html.

Fill in a photo

  1. Find the photo — use the Search box at the top or the folder dropdown; recognise it by its thumbnail.
  2. Type into the fields you want. Camera settings are already filled from EXIF (marked “auto”). Add: Title, Collection (pick from the dropdown), Location, Coordinates, Story, Challenges, Keywords, and the Featured tick if it's a highlight.
  3. Click Generate (or Copy) at the bottom — it produces the saved data.
  4. Get that data into the site: either Download it (it saves photo-meta.js, which replaces js/photo-meta.js), or hand the copied text to whoever publishes.
Coverage dashboard: the top of the editor shows how many photos have titles, GPS, stories, etc. Use the filters (“Missing story”, “No camera data”) to work through a big library in batches.

Good habits

  • Camera fields auto-tidy as you type (9.1 → f/9.1, 2000 → ISO 2000) — type naturally.
  • Your edits are safe forever — re-running the build never overwrites them.

How-to

Locations & map coordinates

A photo with coordinates gets a map on its story page and a pin on the Photo Map.

Getting coordinates

  • Easiest: In Google Maps, right-click the exact spot → click the numbers at the top to copy them (e.g. 6.003167, 80.256556).
  • If your photo already has GPS from the camera/phone, it appears automatically — nothing to do.

Two formats — both accepted

The editor's “Coordinates” field accepts either form and fills the latitude/longitude for you:

  • Decimal: 6.003167, 80.256556
  • DMS (degrees-minutes-seconds): 6°00'11.4"N 80°15'23.6"E — it converts to decimal automatically.
Note: the Latitude/Longitude boxes themselves expect plain decimal numbers — but the “Coordinates” paste field handles the conversion, so just paste whatever you have there.

Organising

Collections — grouping photos into stories

A collection is a themed set (e.g. “Desert Adventures”, “Night Skies”). They appear in a Collections section at the bottom of the Gallery page, and each opens a view of just those photos.

How to create one

There's nothing to “create” — just give photos the same Collection name in the editor's Collection field, and the collection appears automatically. The dropdown suggests a ready-made list.

  • One photo belongs to one collection.
  • The Featured photo (if any) becomes the collection's cover image.
  • Use Keywords for cross-cutting tags (like place names) — those feed search, not collections.
Suggested theme set: Night Skies · Golden Hours · Cityscapes · Desert Adventures · Coast & Sea · Highlands & Rails · From Above · Fine Art · Faces · On the Wing.

The flagship feature

Photo Stories

Every photo has its own story page: a large image (or live 360 view), the title, location, your story and challenges, the camera settings, a location map, and related images.

How visitors reach a story

  • From the gallery: open a photo → click the small “Read the full story” button.
  • From the map: click a pin → “View full story”.
  • From search results: click any thumbnail.

Writing a good story

  • Story: what the moment was, why it matters, what's happening. A short paragraph or two.
  • Challenges: the effort behind it — the trek, the cold, the timing. People love this.
  • Leave a blank line between paragraphs and they'll format nicely.
Tip: a photo with camera data and a location already makes a decent page even without a written story — but the story is what turns a portfolio into a destination.

Feature

The interactive map

The Map page (in the menu) pins every photo that has coordinates. Pins cluster when zoomed out; click one to see the photo, then open its story. A 🌍 reset button (under the zoom + / −) returns to the full view.

How to add a pin

Just give a photo coordinates (see the Locations lesson). On the next publish, its pin appears — no map editing needed. 360 photos open in the panorama viewer right from the map.

Feature

Search

On the Gallery page, below the galleries, is a search box. It finds photos by keywords, title, location, country, collection, gallery name, and camera. All typed words must match (so “Kuwait night” narrows precisely).

Making photos findable

  • Every photo is searchable by its gallery name the moment it's uploaded.
  • Add Keywords (in Lightroom or the editor) to make it findable by subject, place, and mood.
  • Think like a visitor: subject (camel, dunes, Milky Way), place (Riyadh, Sri Lanka), mood (sunset, foggy, blue hour). 5–10 keywords per photo.

Feature

360° panoramas

Equirectangular 360 photos go in the photos_360 folder. They open in a drag-to-look viewer in the gallery, on the map, and on their story page, and they rotate live in the home hero.

Setting a 360's location

360 photos often lack GPS, so their map location is set by hand in js/pano-locations.js, keyed by filename. To set or change one, ask the developer (or give them the coordinates) — same decimal/DMS rules apply.

Going live

Publishing your changes

Nothing is live until it's published. There are two routes depending on how you're set up.

Route A — You publish it yourself

In a terminal in the project folder:

git add -A git commit -m "Add new photos and details" git push

Netlify sees the change and rebuilds the live site automatically in ~1–2 minutes.

Route B — Hand it over

If you'd rather not use the terminal, give the changed files (or the copied photo-meta.js text from the editor) to whoever manages the site, and they'll publish it.

Check it went live

  1. Wait ~2 minutes.
  2. Open the site and press Ctrl+Shift+R (a “hard refresh”) to bypass the cache.
  3. Confirm your change shows.
If a change doesn't appear, it's almost always the browser cache — hard-refresh, or wait a couple more minutes for the edge to update.

Discoverability

SEO & Google

Most SEO is automatic. The build creates a sitemap (a list of every page, including each photo's story) and the pages carry share-cards (so links show a preview image) and structured data for Google.

What you do

  • Once: submit the sitemap in Google Search Console (Sitemaps → enter sitemap.xml → Submit).
  • Ongoing: write good titles, locations, and keywords — that's what Google indexes. The sitemap refreshes itself on every build.
When the site moves to its own domain (e.g. liophotography.com), the developer updates one setting (SITE_URL) so share-cards and the sitemap point to the new address.

Common questions

My change isn't showing

Hard-refresh (Ctrl+Shift+R), wait 2 minutes, and confirm it was published (pushed to GitHub).

A photo has no camera info / no map

Its file was exported without metadata. Re-export from Lightroom with “All Metadata”, or type the details in the editor.

The 360 viewer is blank when I open a file directly

360 needs the site served over the web (not opened as a local file). Use the live site, or the local helper to serve it.

When in doubt, write down exactly what you did and what you expected, and send it to the developer — that's the fastest way to get unstuck.
